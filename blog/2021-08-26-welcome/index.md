The Digital Clock Prototype was developed as a core assessment for the Product Development Course, demonstrating the full lifecycle of creating an embedded system. Leading a team of five mechatronics students, I coordinated the firmware development using the Mbed OS framework—specifically leveraging Timer and Ticker APIs—to program an NXP KL25Z development board powered by an ARM Cortex-M0+ processor.

Read the full project page here: [Digital Clock project page](/eadwine/docs/academic-projects/04-digital-clock).

### Project Scope

My role encompassed end-to-end ownership, from initial code architecture and task delegation to hands-on PCB soldering, system debugging, and iterative testing. We successfully transformed the initial concept into a fully functional product prototype, navigating challenges such as electrical faults and firmware timing issues. This project served as a practical application of embedded programming, hardware integration, and agile problem-solving in a team environment.

### Key Responsibilities

- **Architecture & Leadership**: Defined the firmware architecture and delegated tasks to team members.
- **Firmware Development**: Developed C/C++ libraries and programmed the digital clock logic using Mbed OS Timer and Ticker APIs.
- **Hardware Integration**: Coordinated PCB design, component selection, and layout for the NXP KL25Z board.
- **Testing & Debugging**: Performed extensive system testing and resolved electrical faults during the soldering and integration phase.
- **Team Coordination**: Managed team workflows, milestones, and dependencies to ensure timely delivery of the prototype.

### What We Learned

- How to structure timer-driven firmware for reliable embedded systems that handle real-time constraints.
- How to systematically debug electrical faults in a hardware prototype and coordinate repair efforts.
- How to coordinate a multi-disciplinary team to move from design concept through to a fully functional product.
- The importance of iterative testing and the value of hands-on problem-solving in embedded product development.
- How to balance theoretical design with practical constraints in a resource-limited academic environment.

This project showcases the integration of firmware architecture, hardware design, and team leadership in delivering a complete embedded system prototype. It remains a testament to what can be achieved through dedication, collaboration, and systematic engineering practices.
