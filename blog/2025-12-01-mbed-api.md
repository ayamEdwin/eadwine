The DarkLight STM32 API project was created to bridge a critical gap in embedded systems education. The retirement of the Mbed OS platform (effective July 2026) created a significant challenge for our curriculum at Koforidua Technical University. The subsequent move to low-level STM32 programming (bare-metal, CMSIS, HAL) proved excessively difficult for sophomore engineering students, hindering their ability to engage with core course material effectively.

Read the full project page here: [DarkLight STM32 API project page](/eadwine/docs/academic-projects/02-mbed-api).

### The Problem & Solution

With Mbed OS retiring, we faced a critical need to provide students with accessible embedded systems education. I led a team of top students in the Mechatronics department to design and develop an educational STM32 API, inspired by the retired Mbed framework, to abstract low-level hardware complexity for sophomore engineering students.

### Project Implementation

- **API Design**: Created wrapper classes similar to ARM Mbed OS that abstract lower-level, more complex hardware interfaces.
- **Target Platform**: Customized the API for the STM32F030RC6 microcontroller, a low-cost ARM Cortex-M0 device ideal for educational purposes.
- **Development Tools**: Used Segger Embedded Studio for development and debugging, and GitHub for version control and collaborative development.
- **API Philosophy**: Designed with a familiar interface similar to ARM Mbed style, reducing the learning curve and allowing students to focus on embedded systems concepts rather than API syntax.
- **Hardware Support**: Carefully exposed essential GPIO, ADC, UART, and timer peripherals while abstracting the complexities of register-level configuration.

### Outcome & Impact

- Students can now prioritize application development over register configuration, focusing on logic rather than hardware manipulation.
- The solution was adopted by university staff and scaled nationally under the name "DarkLight."
- The API continues to aid embedded systems education across Ghana, bridging the gap left by Mbed OS retirement.
- The project demonstrates how educational tools can be designed to remove barriers to learning while maintaining technical rigor.

### What This Means for Education

This project enabled us to maintain quality embedded systems education despite a major industry shift. By creating a student-friendly abstraction layer, we empowered sophomore students to apply their knowledge to real problems rather than struggling with low-level hardware configuration. The DarkLight platform is now a testament to the importance of educational innovation in response to industry changes.
